Tributary

Tributary is an experimental environment for rapidly prototyping visualization code. The environment provides several libraries useful for this kind of coding, as well as a simple interface for live code editing.

Sharing

The goal of Tributary is to reduce the friction involved in creating something visual with code and sharing the result. Once you have created something it should also be effortless to share, just click the fork button and share the new URL with anyone. When you don't have to worry about saving and tracking files, it becomes much easier to let go and get creative, the fork button saves the code into a gist where you can always access it.

Features

Tributary comes with jQuery, d3, underscore and backbone available in the code editor. It is also possible to use CoffeeScript. The editor is powered by CodeMirror. The Bret Victor-like interactive slider and color picker are implemented in Inlet and powered by jQuery UI and ColorPicker

Attribution

Tributary is possible thanks to much harder work done by other people. We started with Gabriel Florit's implementation of concepts introduced in Bret Victor's talk. The use of gists to drive the code in the view comes from Mike Bostock's bl.ocks.org. Tributary is primarily the work of Ian Johnson and EJ Fox, with contributions from Kai Chang and Nils Scholman.
